# MarketMirrorAI Visual Patterns



Purpose:



Store visual evidence for each MMAI pattern.



Structure:



Each MMAI folder contains:



- book_examples

- live_examples



Book Examples:



Images extracted from the author's books.



Live Examples:



Charts generated from market OHLCV data using APIs such as Finnhub.



Operational Rule:



OpenClaw must compare textual rules with visual evidence before producing CALL, PUT, WAIT, or EXIT.



Important:



Text-only reasoning is not sufficient when the author explains a concept using charts.
